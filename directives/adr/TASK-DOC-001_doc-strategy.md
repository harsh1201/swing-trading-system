# ADR-DOC-001: Documentation Strategy
**Date:** 2026-03-26
**Status:** implemented

## Problem Statement
The current documentation is minimal and doesn't provide enough information for developers to set up, understand, and extend the system.

## Proposed Strategy
We will create a multi-layered documentation structure:
1.  **Revamped README.md**: High-level overview, quick start, architecture diagram, and key features.
2.  **docs/installation.md**: Detailed setup guide, environment variables, and dependency management.
3.  **docs/architecture.md**: Technical breakdown of the system components and data flow.
4.  **docs/usage_guide.md**: How to run the backtest, screener, and interpret results.

## Rationale
Separating documentation into specific topics ensures that users can find the information they need without being overwhelmed.

## Definition of Done
- `README.md` is comprehensive and visually appealing.
- Detailed technical documentation is created in the `docs/` folder.
